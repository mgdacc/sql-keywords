Cheat Sheet de SQL

Keywords

1 / 22

¿Qué es una "Keyword"?

Son palabras reservadas que SQL entiende como órdenes o instrucciones. Le dicen a la base de datos QUÉ hacer (SELECT, UPDATE) y DÓNDE hacerlo (FROM, WHERE).

2 / 22

DML: Manipulación de Datos

El grupo de comandos más común, usado para interactuar con los datos existentes. Son las operaciones del día a día.

SELECT, INSERT, UPDATE, DELETE

3 / 22

SELECT y FROM

SELECT especifica las columnas que quieres ver y FROM la tabla de donde las obtendrás. Es la base de toda consulta.

-- Selecciona el nombre y email de todos los usuarios
SELECT Nombre, Email FROM Usuarios;
4 / 22

WHERE: Filtrado de Datos

Permite establecer condiciones para filtrar los registros y obtener solo los que cumplen ciertos criterios.

-- Selecciona los productos que cuestan más de 50
SELECT * FROM Productos WHERE Precio > 50;
5 / 22

Operadores Lógicos

Se usan con WHERE para combinar múltiples condiciones: AND (ambas deben ser ciertas), OR (al menos una debe ser cierta).

-- Productos caros Y de la categoría 'Electrónica'
SELECT * FROM Productos 
WHERE Precio > 1000 AND Categoria = 'Electrónica';
6 / 22

Otros Operadores de Filtrado

IN (está en una lista), BETWEEN (está en un rango), LIKE (coincide con un patrón de texto).

-- Pedidos realizados en Enero o Febrero
SELECT * FROM Pedidos 
WHERE MONTH(Fecha) IN (1, 2);
7 / 22

ORDER BY: Ordenar Resultados

Clasifica los resultados de una consulta en orden ascendente (ASC, por defecto) o descendente (DESC).

-- Empleados ordenados por salario, de mayor a menor
SELECT Nombre, Salario FROM Empleados
ORDER BY Salario DESC;
8 / 22

INSERT INTO: Añadir Registros

Agrega una nueva fila (o registro) a una tabla existente.

-- Agrega un nuevo departamento
INSERT INTO Departamentos (ID, Nombre)
VALUES (5, 'Marketing');
9 / 22

UPDATE y SET: Modificar

UPDATE especifica la tabla y SET las columnas a cambiar. Se usa con WHERE para no modificar toda la tabla.

-- Actualiza el stock del producto con ID 123
UPDATE Productos
SET Stock = 50
WHERE ProductoID = 123;
10 / 22

DELETE: Eliminar Registros

Borra filas de una tabla. ¡Es crucial usar WHERE para especificar qué filas eliminar!

-- Elimina las órdenes canceladas
DELETE FROM Pedidos
WHERE Estado = 'Cancelado';
11 / 22

DDL: Definición de Datos

Este grupo de comandos no trabaja con los datos, sino con la estructura de la base de datos: las tablas, columnas, etc.

CREATE, ALTER, DROP

12 / 22

CREATE TABLE: Crear Estructuras

Define y crea una nueva tabla, especificando sus columnas y los tipos de datos que almacenarán (texto, número, fecha, etc.).

CREATE TABLE Clientes (
    ClienteID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    FechaRegistro DATE
);
13 / 22

ALTER TABLE: Modificar Tablas

Permite modificar una tabla existente, por ejemplo, para añadir, eliminar o cambiar una columna.

-- Añade una columna para el teléfono del cliente
ALTER TABLE Clientes
ADD Telefono VARCHAR(20);
14 / 22

DROP TABLE: Eliminar Tablas

Elimina una tabla completa, incluyendo su estructura y todos sus datos. Es una acción irreversible.

-- Elimina una tabla que ya no se necesita
DROP TABLE Empleados_Antiguos;
15 / 22

Funciones de Agregación

Realizan un cálculo sobre un conjunto de valores y devuelven un único valor. Ej: COUNT, SUM, AVG, MAX, MIN.

-- Calcula el precio promedio de todos los productos
SELECT AVG(Precio) FROM Productos;
16 / 22

GROUP BY: Agrupar Datos

Agrupa filas que tienen los mismos valores en columnas específicas, para poder ejecutar funciones de agregación en cada grupo.

-- Cuenta cuántos productos hay en cada categoría
SELECT Categoria, COUNT(*) FROM Productos
GROUP BY Categoria;
17 / 22

HAVING: Filtrar Grupos

Es como un WHERE, pero para los resultados de un GROUP BY. Filtra grupos, no filas individuales.

-- Muestra categorías con más de 10 productos
SELECT Categoria, COUNT(*) FROM Productos
GROUP BY Categoria
HAVING COUNT(*) > 10;
18 / 22

JOIN: Unir Tablas

Permite combinar filas de dos o más tablas basándose en una columna relacionada entre ellas (como un ID).

-- Une Clientes con Pedidos para ver quién pidió qué
SELECT Clientes.Nombre, Pedidos.Producto
FROM Clientes
JOIN Pedidos ON Clientes.ClienteID = Pedidos.ClienteID;
19 / 22

Tipos de JOIN

INNER JOIN: Solo filas que coinciden en ambas tablas.
LEFT JOIN: Todas las filas de la tabla izquierda y las coincidencias de la derecha.
RIGHT JOIN: Lo opuesto a LEFT JOIN.

-- Muestra todos los clientes, tengan o no pedidos
SELECT C.Nombre, P.Producto
FROM Clientes AS C
LEFT JOIN Pedidos AS P ON C.ClienteID = P.ClienteID;
20 / 22

AS: Alias

Permite renombrar temporalmente una columna o una tabla en una consulta para hacerla más corta o legible.

-- Renombra la columna y la tabla para más claridad
SELECT Nombre AS NombreCompleto
FROM Empleados AS E;
